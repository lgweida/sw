Tidal Automation Software Overview and Features
do you know tidal automation software?

Of course. Yes, I am very familiar with Tidal Automation Software. It's a major player in the world of enterprise job scheduling and workload automation.

Here’s a comprehensive breakdown of what Tidal is, based on its history and current state.

What is Tidal Automation?

At its core, Tidal Automation is a software platform designed to automate, schedule, and manage business processes and IT workloads. Think of it as a highly sophisticated, centralized conductor for a company's critical computing tasks.

It ensures that the right jobs run at the right time, in the correct order, and across diverse systems—from mainframes and servers to cloud applications.

Key Historical Context: From Cisco to Redwood

The name "Tidal" can be a bit confusing because of its history:

  1. Tidal Software (Independent Company): The company was originally founded as Tidal Software and was a well-respected competitor in the job scheduling market.

  2. Acquisition by Cisco (2009): Cisco Systems acquired Tidal Software to bolster its data center and cloud management offerings.

  3. Acquisition by Redwood Software (2023): This is the most critical recent development. Redwood Software, a leader in the RunMyJob robotic process automation and workload automation space, acquired the Tidal Automation product from Cisco.

Today, when people refer to "Tidal Automation," they are almost always talking about the product now owned and developed by Redwood.

Core Capabilities and Features

Tidal's power comes from its ability to handle complex, cross-platform workflows. Its main features include:

  • Cross-Platform Scheduling: It can manage jobs on a vast array of systems, including:

    • Operating Systems: Windows, Linux, Unix (AIX, HP-UX, Solaris)

    • ERP Systems: SAP, Oracle E-Business Suite

    • Databases: Microsoft SQL Server, Oracle, DB2

    • Applications: PeopleSoft, Siebel, custom .NET and Java apps

    • Mainframes: Can integrate with and coordinate jobs on IBM z/OS.

    • Cloud & Containers: AWS, Azure, Google Cloud Platform, Kubernetes

  • Workflow Orchestration: This is its primary strength. You can build complex workflows where the success or failure of one job triggers the next, sends alerts, or branches to different paths. For example, "After the nightly sales data ETL job finishes successfully, run the database consolidation process, and then, only if both succeed, trigger the reporting package."

  • Centralized Management and Monitoring: Provides a single pane of glass to view the status of all automated processes across the entire enterprise. This makes it easy to see what's running, what succeeded, and what failed.

  • Calendar-Based Scheduling: Beyond simple time-based scheduling (e.g., "run at 2 AM"), it supports sophisticated calendar rules like "run on the last business day of the month" or "run on the third Monday of every quarter."

  • Robust Error Handling and Alerting: It can be configured to automatically retry failed jobs, run cleanup scripts, and send detailed alerts via email, SMS, or SNMP to on-call staff.

  • Integration and APIs: Offers APIs (often RESTful) to integrate with other IT management tools, service desks (like ServiceNow), and custom applications.

Who Uses Tidal Automation?

It's primarily an enterprise-grade solution used by large organizations in industries where business processes are critical and complex:

  • Financial Services: For end-of-day trading, risk calculation, and regulatory reporting.

  • Manufacturing: For supply chain management, order processing, and inventory updates.

  • Healthcare: For patient billing systems, data backups, and report generation.

  • Retail: For processing online orders, updating inventory, and analyzing sales data.

Typical users are IT Operations teams, DevOps engineers, and application owners.

Tidal Automation vs. Competitors

Tidal/Redwood competes in the "Workload Automation" space against other major players like:

  • Broadcom (formerly CA Automic): Another heavyweight, especially strong in mainframe and hybrid environments.

  • IBM Workload Scheduler (IWS): Often used in shops heavily invested in the IBM ecosystem.

  • Stonebranch: Known for its modern, container-native Universal Automation Center platform.

  • BMC Control-M: A direct and very strong competitor with a similar feature set and market focus.

Summary

In short, Tidal Automation is a powerful, enterprise-class workload automation tool designed to bring order and reliability to complex IT processes. Its acquisition by Redwood has positioned it as a key component of a larger automation strategy, often integrating with robotic process automation (RPA) and other modern IT orchestration needs.

import requests
import json
from typing import List, Dict, Optional

class TidalAutomationClient:
    def __init__(self, base_url: str, username: str, password: str, verify_ssl: bool = True):
        """
        Initialize Tidal Automation REST API client
        
        Args:
            base_url: Base URL of your Tidal instance (e.g., "https://your-tidal-server.com")
            username: Your username
            password: Your password
            verify_ssl: Whether to verify SSL certificates
        """
        self.base_url = base_url.rstrip('/')
        self.username = username
        self.password = password
        self.verify_ssl = verify_ssl
        self.session = requests.Session()
        
        # For basic auth (common in Tidal APIs)
        self.session.auth = (username, password)
        
        # Alternatively, for token-based auth if your Tidal instance uses it
        # self.token = self.authenticate()
    
    def authenticate(self) -> Optional[str]:
        """
        Authenticate and get token (if using token-based auth)
        """
        auth_url = f"{self.base_url}/api/authenticate"
        auth_data = {
            'username': self.username,
            'password': self.password
        }
        
        try:
            response = self.session.post(
                auth_url,
                json=auth_data,
                verify=self.verify_ssl
            )
            response.raise_for_status()
            
            # Extract token from response (adjust based on actual API response)
            auth_response = response.json()
            return auth_response.get('token')
            
        except requests.exceptions.RequestException as e:
            print(f"Authentication failed: {e}")
            return None
    
    def get_my_jobs(self, created_by: str = None) -> List[Dict]:
        """
        Get all jobs created by the current user
        
        Args:
            created_by: Optional filter for creator username
                       If None, will use current username
        
        Returns:
            List of job dictionaries
        """
        if created_by is None:
            created_by = self.username
        
        # API endpoint - this may vary based on your Tidal version
        jobs_url = f"{self.base_url}/api/jobs"
        
        try:
            # Add query parameters to filter by creator
            params = {
                'createdBy': created_by
                # You might also want to add:
                # 'pageSize': 1000,  # Adjust as needed
                # 'fields': 'id,name,description,status,createdBy,createdDate'
            }
            
            response = self.session.get(
                jobs_url,
                params=params,
                verify=self.verify_ssl
            )
            response.raise_for_status()
            
            jobs_data = response.json()
            
            # The structure might vary - adjust based on actual API response
            if isinstance(jobs_data, dict) and 'jobs' in jobs_data:
                return jobs_data['jobs']
            elif isinstance(jobs_data, dict) and 'items' in jobs_data:
                return jobs_data['items']
            elif isinstance(jobs_data, list):
                return jobs_data
            else:
                print(f"Unexpected response format: {jobs_data}")
                return []
                
        except requests.exceptions.RequestException as e:
            print(f"Failed to fetch jobs: {e}")
            return []
    
    def get_all_jobs_with_filter(self) -> List[Dict]:
        """
        Alternative approach: Get all jobs and filter locally
        Useful if the API doesn't support filtering by creator
        """
        jobs_url = f"{self.base_url}/api/jobs"
        
        try:
            response = self.session.get(jobs_url, verify=self.verify_ssl)
            response.raise_for_status()
            
            all_jobs = response.json()
            
            # Filter jobs created by current user
            my_jobs = [
                job for job in all_jobs 
                if job.get('createdBy', '').lower() == self.username.lower()
            ]
            
            return my_jobs
            
        except requests.exceptions.RequestException as e:
            print(f"Failed to fetch jobs: {e}")
            return []

class EnhancedTidalClient:
    def __init__(self, base_url: str, username: str, password: str, verify_ssl: bool = True):

        self.username = username

        self.verify_ssl = verify_ssl
        
        # Add common headers
        self.session.headers.update({
            'Content-Type': 'application/json',
            'Accept': 'application/json'
        })
    
    def test_connection(self) -> bool:
        """Test if we can connect to the Tidal API"""
        try:
            test_url = f"{self.base_url}/api/status"
            response = self.session.get(test_url, verify=self.verify_ssl, timeout=10)
            return response.status_code == 200
        except:
            return False
    
    def get_my_jobs_enhanced(self, max_results: int = 1000) -> List[Dict]:
        """
        Enhanced method to get jobs with better error handling and pagination
        """
        jobs = []
        page = 1
        page_size = min(100, max_results)
        
        while len(jobs) < max_results:
            params = {
                'createdBy': self.username,
                'page': page,
                'pageSize': page_size
            }
            
            try:
                response = self.session.get(
                    f"{self.base_url}/api/jobs",
                    params=params,
                    verify=self.verify_ssl,
                    timeout=30
                )
                
                if response.status_code == 404:
                    # Try different endpoint or approach
                    return self._fallback_job_search()
                
                response.raise_for_status()
                data = response.json()
                
                # Handle different response formats
                page_jobs = self._extract_jobs_from_response(data)
                
                if not page_jobs:
                    break  # No more jobs
                
                jobs.extend(page_jobs)
                
                # Check if we've reached the last page
                if len(page_jobs) < page_size:
                    break
                    
                page += 1
                
            except requests.exceptions.RequestException as e:
                print(f"Error fetching page {page}: {e}")
                break
        
        return jobs[:max_results]
    
    def _extract_jobs_from_response(self, data: dict) -> List[Dict]:
        """Extract jobs from various possible response formats"""
        if isinstance(data, list):
            return data
        elif isinstance(data, dict):
            for key in ['jobs', 'items', 'results', 'content']:
                if key in data and isinstance(data[key], list):
                    return data[key]
        return []
    
    def _fallback_job_search(self) -> List[Dict]:
        """Fallback method if primary endpoint fails"""
        print("Using fallback method to search for jobs...")
        try:
            response = self.session.get(
                f"{self.base_url}/api/jobs",
                verify=self.verify_ssl
            )
            response.raise_for_status()
            
            all_jobs = self._extract_jobs_from_response(response.json())
            return [job for job in all_jobs if job.get('createdBy') == self.username]
            
        except requests.exceptions.RequestException as e:
            print(f"Fallback method also failed: {e}")
            return []

Here's a comprehensive Python script to export and import job definitions in Tidal Automation using REST API:

Complete Export/Import Script

python
import requests
import json
import os
import yaml
import argparse
from pathlib import Path
from typing import Dict, List, Optional
import time

class TidalJobManager:
    def __init__(self, base_url: str, username: str, password: str, verify_ssl: bool = True):
        self.base_url = base_url.rstrip('/')
        self.username = username
        self.session = requests.Session()
        self.session.auth = (username, password)
        self.verify_ssl = verify_ssl
        
        self.session.headers.update({
            'Content-Type': 'application/json',
            'Accept': 'application/json'
        })
    
    def export_job(self, job_id: str, export_format: str = 'json') -> Optional[Dict]:
        """
        Export a job definition
        
        Args:
            job_id: The ID of the job to export
            export_format: Format to export ('json' or 'yaml')
            
        Returns:
            Job definition dictionary or None if failed
        """
        try:
            # Endpoint to get job definition - may vary by Tidal version
            job_url = f"{self.base_url}/api/jobs/{job_id}"
            
            response = self.session.get(job_url, verify=self.verify_ssl)
            
            if response.status_code == 404:
                print(f"Job {job_id} not found")
                return None
            
            response.raise_for_status()
            job_definition = response.json()
            
            # Clean sensitive data or add metadata
            cleaned_definition = self._clean_job_definition(job_definition)
            
            return cleaned_definition
            
        except requests.exceptions.RequestException as e:
            print(f"Failed to export job {job_id}: {e}")
            return None
    
    def export_job_to_file(self, job_id: str, output_path: str, export_format: str = 'json'):
        """
        Export job definition to file
        """
        job_definition = self.export_job(job_id, export_format)
        
        if job_definition:
            # Ensure directory exists
            Path(output_path).parent.mkdir(parents=True, exist_ok=True)
            
            with open(output_path, 'w', encoding='utf-8') as f:
                if export_format.lower() == 'yaml':
                    yaml.dump(job_definition, f, default_flow_style=False, indent=2)
                else:
                    json.dump(job_definition, f, indent=2, ensure_ascii=False)
            
            print(f"✅ Job {job_id} exported to {output_path}")
            return True
        else:
            print(f"❌ Failed to export job {job_id}")
            return False
    
    def export_my_jobs(self, output_dir: str, export_format: str = 'json') -> List[str]:
        """
        Export all jobs created by current user
        """
        exported_files = []
        
        # Get all jobs created by current user
        jobs = self.get_my_jobs()
        
        if not jobs:
            print("No jobs found to export")
            return exported_files
        
        # Create output directory
        Path(output_dir).mkdir(parents=True, exist_ok=True)
        
        print(f"Exporting {len(jobs)} jobs to {output_dir}...")
        
        for job in jobs:
            job_id = job.get('id')
            job_name = job.get('name', 'unknown').replace('/', '_').replace('\\', '_')
            
            if job_id:
                filename = f"{job_name}_{job_id}.{export_format}"
                output_path = os.path.join(output_dir, filename)
                
                if self.export_job_to_file(job_id, output_path, export_format):
                    exported_files.append(output_path)
            
            # Small delay to avoid overwhelming the API
            time.sleep(0.1)
        
        print(f"✅ Exported {len(exported_files)} jobs")
        return exported_files
    
    def import_job(self, job_definition: Dict, new_job_name: str = None) -> Optional[Dict]:
        """
        Import a job definition to create a new job
        
        Args:
            job_definition: The job definition dictionary
            new_job_name: Optional new name for the job
            
        Returns:
            Created job data or None if failed
        """
        try:
            # Prepare the job definition for import
            import_definition = self._prepare_for_import(job_definition, new_job_name)
            
            # Endpoint to create job
            create_url = f"{self.base_url}/api/jobs"
            
            response = self.session.post(
                create_url,
                json=import_definition,
                verify=self.verify_ssl
            )
            
            if response.status_code in [200, 201]:
                created_job = response.json()
                print(f"✅ Job created successfully: {created_job.get('name')} (ID: {created_job.get('id')})")
                return created_job
            else:
                print(f"❌ Failed to create job: {response.status_code} - {response.text}")
                return None
                
        except requests.exceptions.RequestException as e:
            print(f"Failed to import job: {e}")
            return None
    
    def import_job_from_file(self, file_path: str, new_job_name: str = None) -> Optional[Dict]:
        """
        Import job definition from file
        """
        try:
            with open(file_path, 'r', encoding='utf-8') as f:
                if file_path.lower().endswith('.yaml') or file_path.lower().endswith('.yml'):
                    job_definition = yaml.safe_load(f)
                else:
                    job_definition = json.load(f)
            
            return self.import_job(job_definition, new_job_name)
            
        except Exception as e:
            print(f"Failed to read job definition from {file_path}: {e}")
            return None
    
    def import_all_jobs_from_directory(self, import_dir: str) -> List[Dict]:
        """
        Import all job definitions from a directory
        """
        imported_jobs = []
        import_dir = Path(import_dir)
        
        # Find all job definition files
        json_files = list(import_dir.glob("*.json"))
        yaml_files = list(import_dir.glob("*.yaml")) + list(import_dir.glob("*.yml"))
        all_files = json_files + yaml_files
        
        if not all_files:
            print(f"No job definition files found in {import_dir}")
            return imported_jobs
        
        print(f"Found {len(all_files)} job definition files. Starting import...")
        
        for file_path in all_files:
            print(f"Importing {file_path.name}...")
            created_job = self.import_job_from_file(str(file_path))
            
            if created_job:
                imported_jobs.append(created_job)
            
            # Small delay to avoid overwhelming the API
            time.sleep(0.2)
        
        print(f"✅ Successfully imported {len(imported_jobs)} jobs")
        return imported_jobs
    
    def _clean_job_definition(self, job_definition: Dict) -> Dict:
        """
        Clean job definition by removing system-generated fields
        """
        # Fields to remove (system-generated or not needed for import)
        fields_to_remove = [
            'id', 'jobId', 'createdDate', 'modifiedDate', 
            'lastRun', 'nextRun', 'executionCount',
            'createdBy', 'modifiedBy', 'version',
            'systemFields', 'internalId'
        ]
        
        cleaned = job_definition.copy()
        
        for field in fields_to_remove:
            if field in cleaned:
                del cleaned[field]
        
        # Add export metadata
        cleaned['_export_metadata'] = {
            'exported_by': self.username,
            'export_date': time.strftime('%Y-%m-%d %H:%M:%S'),
            'export_tool': 'Tidal Job Manager'
        }
        
        return cleaned
    
    def _prepare_for_import(self, job_definition: Dict, new_job_name: str = None) -> Dict:
        """
        Prepare job definition for import
        """
        import_def = job_definition.copy()
        
        # Remove export metadata if present
        if '_export_metadata' in import_def:
            del import_def['_export_metadata']
        
        # Update job name if specified
        if new_job_name and 'name' in import_def:
            import_def['name'] = new_job_name
        
        # Ensure required fields
        if 'name' not in import_def:
            import_def['name'] = f"Imported_Job_{int(time.time())}"
        
        # Set creator to current user
        import_def['createdBy'] = self.username
        
        return import_def
    
    def get_my_jobs(self) -> List[Dict]:
        """
        Get all jobs created by current user (from previous script)
        """
        try:
            jobs_url = f"{self.base_url}/api/jobs"
            params = {'createdBy': self.username}
            
            response = self.session.get(jobs_url, params=params, verify=self.verify_ssl)
            response.raise_for_status()
            
            data = response.json()
            
            # Handle different response formats
            if isinstance(data, dict) and 'jobs' in data:
                return data['jobs']
            elif isinstance(data, dict) and 'items' in data:
                return data['items']
            elif isinstance(data, list):
                return data
            else:
                return []
                
        except requests.exceptions.RequestException as e:
            print(f"Failed to fetch jobs: {e}")
            return []

def create_job_from_template(self, template_name: str, parameters: Dict, new_job_name: str) -> Optional[Dict]:
    """
    Create a job from a template with parameter substitution
    """
    try:
        # Load template
        template_path = f"./job_templates/{template_name}.json"
        with open(template_path, 'r') as f:
            template = json.load(f)
        
        # Replace parameters in template
        job_definition = self._apply_template_parameters(template, parameters)
        job_definition['name'] = new_job_name
        
        return self.import_job(job_definition)
        
    except Exception as e:
        print(f"Failed to create job from template: {e}")
        return None

def _apply_template_parameters(self, template: Dict, parameters: Dict) -> Dict:
    """
    Apply parameter substitution in template
    """
    import json
    template_str = json.dumps(template)
    
    for key, value in parameters.items():
        placeholder = f"${{{key}}}"
        template_str = template_str.replace(placeholder, str(value))
    
    return json.loads(template_str)
